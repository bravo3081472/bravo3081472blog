NVM ( Node.js 版本控制 )

什麼是 NVM

NVM 簡單來說就是 Node.js 的版本管理工具，可以藉由 NVM 來安裝不同版本的 Node.js 並且隨自己的意思自由切換。

NVM 版本切換原理

NVM 是透過捷徑的方式去切換版本的，所以切換版本後，各自的全域套件會在各自的版本目錄中，以至於切換到新的版本都需要重新安裝，避免因為 node 版本套件之間的衝突。

安裝時會要你設定 NVM 跟 node.js 的位置

  • NVM 下載的版本都是放在 NVM 的資料夾內

    預設在此位置：C:\Users\使用者名稱\AppData\Roaming\nvm

  • 下載在此位置的版本透過捷徑的方式，連到 node.js 的路徑裡

    預設在此位置：C:\Program Files\node.js ( 此位置為捷徑 )

常見問題

  1. 為什麼無法使用 NPM ?
    因為一般 NPM 會被包含在 Node.js 裡面，可是透過 NVM 下載的 Node.js 是不包含 NPM 套件的。

    NPM 有需要可以透過這的位置去抓 https://github.com/npm/cli/releases/tag/v6.14.16 ，抓到 nvm 下載的 node_modules 中改名為 npm，複製 bin 中的 npm npm.cmd npx npx.cmd 到與 node.exe 同層的位置或去官網抓舊版本。

  2. nvm use 指定版本 exit status 5
